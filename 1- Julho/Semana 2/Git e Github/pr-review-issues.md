# 📚 - Pull Requests, Code Review e Issues

## 🎯 Objetivo

Entender o fluxo profissional utilizado por equipes de desenvolvimento.

---

## 🧠 Pull Request (PR)

Um Pull Request é um pedido para integrar alterações em outra branch.

Normalmente:

```text
feature-login
↓
Pull Request
↓
main
```

---

## 🧠 Code Review

Processo onde outro desenvolvedor revisa seu código.

Objetivos:

- Encontrar bugs.
- Melhorar legibilidade.
- Garantir padrões da equipe.

---

## 🧠 Issues

Issues representam:

- Bugs
- Melhorias
- Funcionalidades
- Tarefas

Exemplo:

```text
#42 Corrigir erro no login
```

---

## 🔄 Fluxo Profissional

```text
Issue
↓
Criar Branch
↓
Desenvolver
↓
Commit
↓
Push
↓
Pull Request
↓
Code Review
↓
Merge
↓
Fechar Issue
```

---

## ⚠️ Erros Comuns

- Abrir PR sem testar.
- Ignorar comentários do review.
- Criar PRs muito grandes.

---

## 📝 Resumo

- PR não é Merge.
- Code Review melhora a qualidade.
- Issues organizam o trabalho da equipe.
