# Terminal e Shell

O terminal é a interface que permite ao usuário interagir com o computador por meio de comandos de texto. Ele captura o que é digitado e exibe as respostas na tela.

O shell é o programa responsável por interpretar os comandos digitados no terminal e executá-los, comunicando-se com o sistema operacional.

## Diferença

- **Terminal:** interface de entrada e saída de texto.
- **Shell:** interpretador de comandos.

## Fluxo de funcionamento

Usuário → Terminal → Shell → Sistema Operacional → Shell → Terminal → Usuário

## Exemplos

### Terminais
- Windows Terminal
- GNOME Terminal
- Konsole
- Ghostty

### Shells
- Bash
- Zsh
- Fish
- PowerShell

# Git e GitHub

O Git é um sistema de controle de versão que permite registrar alterações em arquivos, acompanhar o histórico de um projeto e voltar para versões anteriores quando necessário.

O GitHub é uma plataforma online que hospeda repositórios Git na nuvem, facilitando o compartilhamento de código, colaboração entre desenvolvedores e criação de portfólios.

## Diferença

- **Git:** ferramenta de versionamento local.
- **GitHub:** plataforma para armazenar e compartilhar repositórios Git.

## Fluxo Básico

1. Criar ou clonar um repositório.
2. Fazer alterações nos arquivos.
3. Adicionar as alterações.
4. Criar um commit.
5. Enviar para o GitHub.

## Comandos Principais

```bash
git clone URL
git status
git add .
git commit -m "mensagem"
git push

```
